# README更新設計

## 概要

プロジェクトのREADMEファイルを更新し、AI開発フローとコマンドシステムの説明を追加する。

## 追加する内容

### 1. AI開発フローセクション

READMEに新しいセクション「AI開発フロー」を追加する。

#### 1.1 フローの概要

`prompts/00-ai-workflow.md`を参照し、開発フローの全体像を説明する。

```
アイデア記録 → 議論 → 議事録記録 → タスク切り出し → 設計 → 実装 → テスト → AIレビュー
```

#### 1.2 コマンド一覧

以下のコマンドの説明を追加する:

- `/idea`: アイデア記録フローに入る
- `/discuss`: 議論フローに入る
- `/design`: 設計フローに入る
- `/implement`: 実装フローに入る
- `/test`: テストフローに入る
- `/review`: AIレビューフローに入る
- `/task`: タスク管理フローに入る
- `/status`: フローの進捗状況を確認
- `/continue`: 直前の作業を再開
- `/policy`: プロジェクトの運用方針を確認

各コマンドの説明は`.cursor/commands/`ディレクトリ内の各コマンドファイルを参照する。

#### 1.3 開発フローの詳細

各ステップの詳細を説明する:

1. **アイデア記録**: `prompts/12-ideas.md`に記録
2. **議論**: アイデアについて議論し、`[議論完了]`に更新
3. **議事録記録**: `prompts/11-meeting-notes.md`に記録
4. **タスク切り出し**: `prompts/14-tasks.md`にタスクを記録
5. **設計**: 設計ドキュメントを作成・更新
6. **実装**: コードを実装
7. **テスト**: テストを作成・実行
8. **AIレビュー**: コードレビューを実施

### 2. プロジェクトの運用方針への参照

#### 2.1 `/policy`コマンドの説明

`/policy`コマンドを使用することで、プロジェクトの運用方針を確認できることを説明する。

#### 2.2 `prompts/`ディレクトリの説明

`prompts/`ディレクトリには以下のドキュメントが含まれることを説明する:

- `00-ai-workflow.md`: AI開発フローの全体像
- `01-project-overview.md`: プロジェクト概要
- `02-architecture.md`: アーキテクチャ設計
- `03-coding-standards.md`: コーディング規約
- `04-directory-structure.md`: ディレクトリ構造
- `05-code-patterns.md`: コードパターン
- `06-type-definitions.md`: 型定義
- `07-testing-strategy.md`: テスト戦略
- `08-development-workflow.md`: 開発フロー
- `10-constraints.md`: 制約事項
- `11-meeting-notes.md`: 議事録
- `12-ideas.md`: アイデア
- `13-api-specifications.md`: API仕様
- `14-tasks.md`: タスク管理
- `15-review-guidelines.md`: レビューガイドライン
