# implement

実装フローを開始するコマンドです。

## 実行内容

1. `prompts/14-tasks.md`を読み込み、設計フェーズが`[完了]`で実装フェーズが`[未着手]`のタスクを確認
2. 実装対象のタスクを特定（`/task`コマンドで選択したタスク、または手動で指定）
3. タスクの実装フェーズを`[進行中]`に更新
4. 設計ドキュメントを確認し、実装を開始する準備をする

## 使用方法

チャットで `/implement` と入力すると、実装フローに入ります。

## 実装時の注意事項

- `14-tasks.md`で設計フェーズが`[完了]`で実装フェーズが`[未着手]`のタスクを確認
- タスクの実装フェーズを`[進行中]`に更新
- 設計ドキュメント（`design/`ディレクトリ内の設計書）を確認
- 既存のコードパターンに従う（`05-code-patterns.md`参照）
- 型定義を適切に使用（`06-type-definitions.md`参照）
- コーディング規約に従う（`03-coding-standards.md`参照）
- 既存ロジックを確認し、重複を避ける
- **API 実装時は必ずフロントエンドとバックエンドの両方を同時に実装する**
- 実装完了後、タスクの実装フェーズを`[完了]`に更新
- 実装完了日を記録

## 次のステップ

実装が完了したら、テストフェーズに進みます。

詳細は `prompts/00-ai-workflow.md` を参照してください。
