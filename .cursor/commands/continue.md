# continue

直前の作業を再開するコマンドです。

## 実行内容

1. 会話履歴を解析し、直前の作業を特定
2. 作業の種類（設計、実装、テスト、AIレビューなど）と内容を表示
3. 作業の続きから再開

## 使用方法

チャットで `/continue` と入力すると、直前の作業を再開します。

## 作業特定の方法

- **同じコンテキスト上の会話のみから特定**: ファイルやタスクステータスは参照しない
- **対象作業**: 設計、実装、テスト、AIレビューなど、会話中に開始された作業
- **特定方法**: 会話履歴から最後に実行していた作業を特定

## 作業の種類

以下のような作業を再開できます:

- **設計**: `/design`コマンドで開始した設計作業
- **実装**: `/implement`コマンドで開始した実装作業
- **テスト**: `/test`コマンドで開始したテスト作業
- **AIレビュー**: `/review`コマンドで開始したAIレビュー作業
- **その他**: 会話中に開始された作業全般

## エラーハンドリング

### 作業を特定できない場合

会話履歴から作業を特定できない場合、以下のメッセージを表示します:

```text
直前の作業を特定できませんでした。
会話履歴に作業の開始記録が見つかりません。
```

### 複数の作業が途中で止まっている場合

複数の作業が途中で止まっている場合、最新の作業を優先して再開します。

## 実行例

### 例1: 設計作業の再開

```text
ユーザー: /design TASK-20250101-002
AI: 設計フローを開始します...
（作業が途中で止まる）

ユーザー: /continue
AI: 直前の作業を特定しました: 設計作業 (TASK-20250101-002)
設計の続きから再開します...
```

### 例2: 実装作業の再開

```text
ユーザー: /implement TASK-20250101-002
AI: 実装フローを開始します...
（作業が途中で止まる）

ユーザー: /continue
AI: 直前の作業を特定しました: 実装作業 (TASK-20250101-002)
実装の続きから再開します...
```

## 注意事項

- `/status`コマンドとの違い:
  - `/status`: フローの進捗状況を確認するコマンド（ファイルベース）
  - `/continue`: 直前の作業を再開するコマンド（会話履歴ベース）
- 会話履歴から作業を特定するため、同じコンテキスト（同じ会話セッション）内でのみ有効
- 作業が完了している場合は、再開する必要はありません
