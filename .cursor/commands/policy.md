# policy

プロジェクトの運用方針を確認するコマンドです。

## 実行内容

1. カテゴリ一覧を表示
2. ユーザーが確認したいカテゴリを選択
3. 選択されたカテゴリに対応するドキュメントを読み込み
4. 運用方針の内容を表示

## 使用方法

チャットで `/policy` と入力すると、プロジェクトの運用方針確認フローに入ります。

## 確認できるカテゴリ

以下のカテゴリから選択できます:

1. **開発フロー**: `prompts/00-ai-workflow.md`の内容
   - AI開発フローの全体像
   - コマンド一覧と使用方法
   - 各フェーズの説明

2. **コーディング規約**: `prompts/03-coding-standards.md`の内容
   - TypeScript設定
   - 命名規則
   - コードスタイル

3. **ディレクトリ構造**: `prompts/04-directory-structure.md`の内容
   - プロジェクトのディレクトリ構造
   - 各ディレクトリの役割
   - ファイル配置規則

4. **テスト戦略**: `prompts/07-testing-strategy.md`の内容
   - テストの方針
   - テストコードの実装タイミング
   - テストカバレッジ

5. **制約事項**: `prompts/10-constraints.md`の内容
   - 使用禁止事項
   - コード重複禁止
   - その他の制約

## 実行フロー

### 1. カテゴリ一覧の表示

以下の形式でカテゴリ一覧を表示します:

```text
## プロジェクトの運用方針

以下のカテゴリから確認したい項目を選択してください:

1. 開発フロー
2. コーディング規約
3. ディレクトリ構造
4. テスト戦略
5. 制約事項

カテゴリ番号を入力してください（1-5）:
```

### 2. カテゴリの選択

ユーザーがカテゴリ番号を入力すると、対応するドキュメントを読み込みます。

### 3. 内容の表示

選択されたカテゴリに対応するドキュメントの内容を表示します。

## 注意事項

- 既存の`/status`コマンドはフローの進捗状況を確認するコマンドであり、運用方針を確認するコマンドとは異なる
- 各カテゴリの内容は`prompts/`ディレクトリ内の該当ドキュメントを参照
- ドキュメントが更新された場合は、最新の内容を表示

## 次のステップ

運用方針を確認した後、開発作業に戻ります。

詳細は `prompts/00-ai-workflow.md` を参照してください。
