# summarize

会話上の流れで行った変更を備考などにまとめるコマンドです。

## 実行内容

1. 会話履歴を解析し、変更内容を抽出
2. 変更内容、修正理由、影響範囲などを構造化された形式でまとめる
3. 議事録やタスクの備考欄に自動的に反映する準備をする

## 使用方法

チャットで `/summarize` と入力すると、会話上の流れで行った変更をまとめます。

## 変更内容の抽出方法

- **同じコンテキスト上の会話のみから抽出**: ファイルやタスクステータスは参照しない
- **対象変更**: 会話中に行った変更や修正
- **抽出方法**: 会話履歴から変更内容を抽出し、構造化された形式でまとめる

## 出力形式

以下の形式で変更内容をまとめます:

```text
## 変更内容のまとめ

### 変更内容
- [変更内容1]
- [変更内容2]

### 修正理由
- [修正理由1]
- [修正理由2]

### 影響範囲
- [影響範囲1]
- [影響範囲2]

### 備考
[その他の情報]
```

## 反映先

- **議事録**: `prompts/11-meeting-notes.md`の備考欄に自動的に反映
- **タスク**: `prompts/14-tasks.md`の備考欄に自動的に反映
- **その他**: 必要に応じて他のドキュメントにも反映

## 注意事項

- `/status`コマンドとの違い:
  - `/status`: フローの進捗状況を確認するコマンド（ファイルベース）
  - `/summarize`: 会話上の流れで行った変更をまとめるコマンド（会話履歴ベース）
- `/continue`コマンドとの違い:
  - `/continue`: 直前の作業を再開するコマンド
  - `/summarize`: 会話上の流れで行った変更をまとめるコマンド
- 会話履歴から変更内容を抽出するため、同じコンテキスト（同じ会話セッション）内でのみ有効
- 変更内容が存在しない場合は、適切なメッセージを表示
