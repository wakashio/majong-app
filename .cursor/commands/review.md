# review

AIレビューフローを開始するコマンドです。

## 実行内容

1. `prompts/14-tasks.md`を読み込み、テストフェーズが`[完了]`でAIレビューフェーズが`[未着手]`のタスクを確認
2. レビュー対象のタスクを特定（`/task`コマンドで選択したタスク、または手動で指定）
3. タスクのAIレビューフェーズを`[進行中]`に更新
4. 実装コード、テストコードをレビューする準備をする
5. コーディング規約の遵守を確認する準備をする
6. 設計ドキュメントとの整合性を確認する準備をする

## 使用方法

チャットで `/review` と入力すると、AIレビューフローに入ります。

## レビュー時の注意事項

- `14-tasks.md`でテストフェーズが`[完了]`でAIレビューフェーズが`[未着手]`のタスクを確認
- タスクのAIレビューフェーズを`[進行中]`に更新
- 実装コードをレビュー（`15-review-guidelines.md`参照）
  - コーディング規約の遵守確認（`03-coding-standards.md`参照）
  - コードパターンの遵守確認（`05-code-patterns.md`参照）
  - 制約事項の確認（`10-constraints.md`参照）
  - 開発フローの遵守確認（`08-development-workflow.md`参照）
- テストコードをレビュー（`15-review-guidelines.md`参照）
  - テストカバレッジの確認
  - テストの品質確認
- 設計ドキュメントとの整合性を確認
  - 設計ドキュメント（`design/`ディレクトリ）との整合性
  - API仕様（`13-api-specifications.md`）との整合性
  - 型定義（`06-type-definitions.md`）との整合性
- 改善点があれば指摘
  - パフォーマンス
  - 可読性
  - 保守性
  - セキュリティ
- レビュー完了後、タスクのAIレビューフェーズを`[完了]`に更新
- レビュー完了日を記録

## レビュー内容

詳細なレビュー内容とチェック項目は `prompts/15-review-guidelines.md` を参照してください。

### 実装コードのレビュー

- コーディング規約の遵守確認
- コードパターンの遵守確認
- 制約事項の確認
- 開発フローの遵守確認

### テストコードのレビュー

- テストコードの実装タイミング確認（TDD）
- テストカバレッジの確認
- テストの品質確認

### 設計ドキュメントとの整合性確認

- 設計ドキュメントとの整合性
- 型定義との整合性
- アーキテクチャとの整合性

### 改善点の指摘

- パフォーマンス
- 可読性
- 保守性
- セキュリティ

## 次のステップ

レビューが完了したら、タスクのAIレビューフェーズを`[完了]`に更新します。

詳細は `prompts/00-ai-workflow.md` と `prompts/15-review-guidelines.md` を参照してください。
