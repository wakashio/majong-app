# task

タスク管理フローを開始するコマンドです。

## 実行内容

1. `prompts/14-tasks.md`を読み込み、タスク一覧を確認
2. 各タスクの 4 フェーズ（設計/実装/テスト/AI レビュー）の進捗を確認
3. タスク管理モードに入ることを明確に示す
4. **実行するタスクを決定**（ユーザーと AI で選択）
5. 選択したタスクの設計フェーズを開始する準備をする

## 使用方法

チャットで `/task` と入力すると、タスク管理フローに入ります。

## タスク選択の流れ

### 1. タスク一覧の表示

以下の情報を表示します:

- タスク ID
- タスクタイトル
- 各フェーズのステータス（設計/実装/テスト/AI レビュー）
- 優先度
- 期限（設定されている場合）

### 2. タスクの優先順位付け

以下の優先順位でタスクを表示します:

1. **最優先**: AI レビューフェーズが`[未着手]`でテストが完了しているタスク
2. **高**: テストフェーズが`[未着手]`で実装が完了しているタスク
3. **高**: 実装フェーズが`[未着手]`で設計が完了しているタスク
4. **高**: 設計フェーズが`[未着手]`のタスク
5. **中**: 各フェーズが`[進行中]`で長時間止まっているタスク

### 3. タスクの決定

- ユーザーと AI で実行するタスクを決定
- 優先度、期限、進捗状況を考慮して選択
- 選択したタスクの情報を確認

### 4. 設計フェーズへの移行

選択したタスクの設計フェーズが`[未着手]`の場合:

- タスクの設計フェーズを`[進行中]`に更新
- `/design`コマンドで設計フローに入ることを提案
- または、直接設計フェーズを開始

選択したタスクの設計フェーズが`[完了]`の場合:

- 次のフェーズ（実装）に進むことを提案

## タスク管理時の注意事項

### タスクの切り出し

- 議事録から実装すべきタスクを特定
- タスク ID を生成（`TASK-YYYYMMDD-XXX`形式）
- 4 フェーズのステータスを`[未着手]`で初期化
- 優先度、期限を設定

### フェーズの進捗管理

- **設計**: 設計ドキュメントへの反映が完了したら`[完了]`に更新
- **実装**: コード実装が完了したら`[完了]`に更新
- **テスト**: テスト作成・実行が完了したら`[完了]`に更新
- **AI レビュー**: コードレビューが完了したら`[完了]`に更新

### フェーズの順序

各フェーズは前のフェーズが完了してから開始することを推奨します:

```text
設計 → 実装 → テスト → AIレビュー
```

## 次のステップ

- タスクの設計フェーズを開始する場合は、`/design`コマンドを使用
- タスクの実装フェーズを開始する場合は、実装を開始
- タスクのテストフェーズを開始する場合は、テストを作成・実行
- タスクの AI レビューフェーズを開始する場合は、コードレビューを実施

詳細は `prompts/00-ai-workflow.md` と `prompts/14-tasks.md` を参照してください。
