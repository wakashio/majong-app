# test

テストフローを開始するコマンドです。

## 実行内容

1. `prompts/14-tasks.md`を読み込み、実装フェーズが`[完了]`でテストフェーズが`[未着手]`のタスクを確認
2. テスト対象のタスクを特定（`/task`コマンドで選択したタスク、または手動で指定）
3. タスクのテストフェーズを`[進行中]`に更新
4. テストを実行し、カバレッジを確認する準備をする
5. 不足しているテストを特定し、作成支援する準備をする

## 使用方法

チャットで `/test` と入力すると、テストフローに入ります。

## テスト時の注意事項

- `14-tasks.md`で実装フェーズが`[完了]`でテストフェーズが`[未着手]`のタスクを確認
- タスクのテストフェーズを`[進行中]`に更新
- テストを実行し、エラーがないことを確認
- テストカバレッジを確認（`07-testing-strategy.md`参照）
- 不足しているテストを特定し、作成支援
- バックエンドの実装には必ずテストを作成（実装フェーズでTDDにより作成済みであることを確認）
- テスト完了後、タスクのテストフェーズを`[完了]`に更新
- テスト完了日を記録

## テストコードの実装タイミングについて

テストコードは実装フェーズでTDD（テスト駆動開発）により先に作成されます。このコマンドは主に以下の役割を果たします：

- テストの実行と確認
- テストカバレッジの確認
- 不足しているテストの特定と作成支援
- テストフェーズの進捗管理

詳細は `prompts/00-ai-workflow.md` と `prompts/07-testing-strategy.md` を参照してください。

## 次のステップ

テストが完了したら、AIレビューフェーズに進みます。

詳細は `prompts/00-ai-workflow.md` を参照してください。
