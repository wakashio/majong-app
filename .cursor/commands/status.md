# status

フローの進捗状況を確認するコマンドです。

## 実行内容

1. `prompts/12-ideas.md`を確認し、途中で止まっているアイデアを検出
2. `prompts/11-meeting-notes.md`を確認し、未反映の議事録を検出
3. `prompts/14-tasks.md`を確認し、途中で止まっているタスクを検出
4. 各フェーズの進捗状況を確認
5. 問題点と次のアクションを提示

## 使用方法

チャットで `/status` と入力すると、フローの進捗状況を確認します。

## 確認項目

### アイデアの確認

- `[議論待ち]`のアイデアが長期間放置されていないか
- `[議論中]`のアイデアが完了していないか
- `[議論完了]`のアイデアが議事録に移行されていないか

### 議事録の確認

- `[未反映]`の議事録がタスクに切り出されていないか
- `[未反映]`の議事録が設計に反映されていないか
- `[反映済み]`の議事録がタスクに切り出されていないか

### タスクの確認

- 設計フェーズが`[進行中]`のまま長時間止まっていないか
- 実装フェーズが`[未着手]`のまま設計が完了していないか
- テストフェーズが`[未着手]`のまま実装が完了していないか
- AIレビューフェーズが`[未着手]`のままテストが完了していないか
- 各フェーズが`[進行中]`のまま長時間止まっていないか

## 出力形式

以下の形式で問題点を報告します:

```text
## フロー進捗状況

### アイデア
- [問題] アイデア名: ステータスと期間

### 議事録
- [問題] 議事録名: ステータスと期間

### タスク
- [問題] タスクID: フェーズ名とステータス、期間

## 推奨アクション
1. 最優先: [具体的なアクション]
2. 高: [具体的なアクション]
...
```

## 次のステップ

問題が検出された場合は、適切なコマンド（`/idea`、`/discuss`、`/task`、`/design`など）を使用して進捗を進めます。

詳細は `prompts/00-ai-workflow.md` を参照してください。
